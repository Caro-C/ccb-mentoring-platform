Wie wird aus meinem Code eine schöne Webseite?

Mai 2020

Level    
           
Home   >   Reads   >   Wie wird aus meinem Code eine schöne Webseite?
Ich hatte dir versprochen, noch einmal genauer zu erzählen, wie aus all dem Code am Ende eine Webseite wird. Code allein ist ja nur Text und dazu auch noch eine ziemlich unleserliche Art von Text. Irgendetwas muss den Code jetzt zum Leben erwecken und das schöne Bild aus ihm machen, das du dir ausgedacht und programmiert hast. Und dieses Ding benutzen wir jedes Mal, wenn wir eine Webseite aufrufen - also vermutlich ständig!
Ich liebe den Moment, in dem ich das, was ich mir ausgedacht und als Code formuliert habe, plötzlich in einem echten Browser sehen kann. Dazu muss meine Webseite noch nicht einmal online sein. Ich kann meine Dateien auch an meinen Browser übergeben, wenn sie ausschließlich auf meinem Laptop gespeichert sind. So kann ich mir das Ganze anschauen, bevor ich es veröffentliche. Das ist ziemlich praktisch, weil ich beim ersten Versuch selten direkt mit dem Ergebnis zufrieden bin.

Nehmen wir aber an, mein Blog ist online und du gibst in deinem Browser www.coffeecodebreak.de ein. Wenn du meinen Artikel Und wie kommt der Blog jetzt ins Internet? gelesen hast, erinnerst du dich vielleicht daran, dass meine Dateien mit meinem Code (und allem, was noch dazugehört) auf einem Webserver liegen. Dein Browser schickt eine Anfrage an diesen Webserver. Der antwortet wiederum, indem er deinem Browser alles schickt, was nötig ist, um meine Webseite anzuzeigen. Das ist nicht nur mein Code, sondern zum Beispiel auch Bilddateien für die Fotos, die du sehen kannst.

Nur weil das Ganze jetzt in deinem Browser angekommen ist, sieht es aber noch nicht schön aus. Im Prinzip sieht es noch überhaupt nicht aus. Irgendetwas muss jetzt noch passieren, damit du meine Webseite mit all ihren Blog-Posts, Fotos und Farben sehen kannst. Und dafür ist ein Programm zuständig, das in deinem Browser enthalten ist. Dieses Programm heißt Rendering Engine. Mit “Rendern” ist nichts anderes gemeint, als aus meinem HTML- und CSS-Code eine visuelle Darstellung zu machen.

Um das mit dem Rendern zu verstehen, müssen wir uns noch einmal mein HTML-Dokument ansehen. Mein HTML-Code hat eine bestimmte Struktur. Diese Struktur setzt sich aus verschiedenen Elementen zusammen, die gemeinsam den Inhalt meiner Seite bilden. Sie sind ineinander verschachtelt, nebeneinander oder nacheinander angeordnet. Zum Beispiel habe ich eine Art Überelement für meine Menüleiste definiert. In diese habe ich Unterelemente gesetzt. Also die Kategorien HOME, READS ..., mein Logo und das LinkedIn-Symbol. Außerdem haben meine HTML-Elemente jeweils einen bestimmten “Charakter”. Manche habe ich zum Beispiel als Überschrift definiert, andere als Paragraph und wieder andere als Bild. Wie genau das mit HTML funktioniert, werde ich dir noch einmal in einem anderem Post erklären.

Wenn ich keinen zusätzlichen CSS-Code geschrieben hätte, wäre meine Webseite nach dem Rendern eine ganz schön hässliche Auflistung von Text und Bildern. Zum Glück hat der Webserver aber auch meinen CSS-Code an deinen Browser geschickt. Meinen Elementen in HTML habe ich bestimmte “Namen” zur Identifizierung gegeben (technisch gesprochen habe ich ihnen Klassen und IDs gegeben). Über diese Namen kann ich mir in CSS ein HTML-Element herauspicken und dieses gestalten. Mein LinkedIn-Symbol wird nur deshalb oben rechts angezeigt, weil ich in CSS definiert habe, dass es dort stehen soll. Und weil du gerade einen Post der Kategorie READS liest, ist READS oben gold eingefärbt. Auch das habe ich mit CSS bestimmt.

Zurück zur Rendering Engine. Die Rendering Engine erkennt die Struktur meines HTML-Codes. Eigentlich zerlegt sie diesen sogar in die einzelnen Elemente und baut die Struktur nochmal selbst nach. Dabei erkennt sie jeweils, um was für ein Element es sich handelt - eine Überschrift, ein Link, ein Bild usw. Und da ich für die Elemente in meinem CSS-Code festgelegt habe, wie sie aussehen und wo sie angeordnet werden sollen, kennt die Rendering Engine diese Information auch. Schlussendlich durchläuft sie ganz schematisch die Struktur und berechnet Element für Element, wo und wie genau es in deinem Bildschirm angezeigt werden soll. Dein Browser kann nun meine Webseite darstellen!

Und so wird aus meinem Code genau der Blog, den du gerade liest. Du siehst also die Visualisierung von dem, was ich mir ausgedacht und mit Code beschrieben habe. Eine Zusammenstellung verschiedener Elemente, die ich nach meinen Vorstellungen angeordnet, gestaltet und mit Inhalt gefüllt habe. Ziemlich faszinierend, oder? Ein Frontend zu programmieren ist also ein bisschen wie malen, nur mit Text!

Vielleicht ist dir aufgefallen, dass ich nur von HTML und CSS gesprochen habe. In meinen anderen Posts hatte ich JavaScript als weitere Sprache erwähnt. Mit JavaScript kann ich meine Seite interaktiv machen. Das kommt aber erst dann richtig zum Zug, wenn meine Seite schon angezeigt wird.

Ich hoffe, der Artikel hat dir gefallen und dich ein bisschen weitergebracht!

Bis bald!
