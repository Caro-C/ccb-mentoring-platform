Wie baut man so eine Webseite?

Mai 2020
Level    
           
Home > Reads > Wie baut man so eine Webseite?
Neulich hat mich meine Mutter gefragt, wie ich denn diesen Blog erstellt hätte. Als ich versucht habe, ihr das zu erklären, habe ich festgestellt, dass ich ziemlich ausschweife. Gar nicht so einfach, das jemandem kurz und knapp zu erklären, der sich kaum mit diesen Themen auskennt. Hier ist also mein zweiter - und dieses Mal schriftlicher - Versuch für euch alle.
Wie habe ich diesen Blog erstellt? Ehrlich gesagt etwas unkonventionell. Da mein Plan war, Programmieren zu üben und die Idee zu diesem Blog erst mit dem Üben kam, habe ich ihn von Grund auf selbst entwickelt. Aber von vorn:

Grundsätzlich gibt es verschiedene Möglichkeiten, eine eigene Webseite zu erstellen. Wenn es dir ausschließlich darum geht, z.B. einen Blog zu starten, kannst du Plattformen wie Wordpress oder Wix nutzen. Hier lassen sich kostenlos und einfach Webseiten erstellen, du benötigst keine Programmierkenntnisse und kannst deine Texte, Bilder und sonstigen Inhalte bequem in einer Datenbank ablegen und verwalten. Es gibt zahlreiche Webseiten und Tutorials, die dir nützliche Tipps geben, wenn du einen solchen Blog erstellen möchtest.

Da es für mich am allerwichtigsten war, meine neu erworbenen Programmierkenntnisse in ein Projekt zu verwandeln, das mir Spaß macht und an dem ich mich ausprobieren und weiterentwickeln kann, ist mein Blog anders entstanden. Hinter all den “Schablonen”, die dir Plattformen wie Wordpress anbieten, steckt natürlich schlussendlich Code. Irgendwer hat diese Schablonen erstellt, indem er den Code dafür geschrieben hat. Das Ergebnis ist, dass du dir eine Schablone aussuchen und deine Inhalte darin so platzieren und gestalten kannst, dass am Ende deine persönliche Webseite dabei herauskommt.

Ich hatte mir also überlegt, was ich bauen könnte, um Programmieren zu üben. Einige frühere Kollegen hatten mir den Tipp gegeben, eine Webseite zu bauen, auf der ich meine Fortschritte festhalten und potenziellen Arbeitgebern zeigen könnte, wie mein aktueller Kenntnisstand ist. Das klang ganz sinnvoll für mich. Also habe ich beschlossen, mir diese Schablone einfach selbst zu bauen und sie danach mit Inhalt zu füllen. Und das hat mir so viel Spaß gemacht, dass daraus am Ende dieser Blog wurde.

Was habe ich also konkret gemacht? Es gibt zahlreiche Programmiersprachen, die für verschiedene Zwecke eingesetzt werden. Um ein sogenanntes “ Web Frontend” zu bauen, also den Teil, den man sehen kann, wenn man eine Webseite aufruft, reichen bereits die drei Sprachen HTML, CSS und JavaScript aus. Eigentlich kannst du sogar mit reinem HTML eine Webseite erstellen. Allerdings würde diese optisch nicht besonders ansprechend ausfallen. Es gibt verschiedene Frameworks und Erweiterungen zu diesen Sprachen, die die Entwicklung einfacher und professioneller machen. Bisher habe ich für diesen Blog ausschließlich HTML, CSS und JavaScript verwendet. Und das hat für den Anfang wunderbar geklappt.

Da ich ein sehr visueller Mensch bin und es liebe, schöne Dinge herzustellen, habe ich mich, als ich begonnen habe, mir das Entwickeln beizubringen, für die Frontend-Entwicklung entschieden. Daher habe ich zunächst genau diese drei Sprachen gelernt. Aber was genau machen nun HTML, CSS und JavaScript und wohin schreibt man überhaupt diesen Code?

Eigentlich ist es erst einmal nichts anderes als eine Ordnerstruktur, die du auf deinem Laptop anlegen kannst und in der du dann Dateien mit deinem Code ablegst. Diese Dateien sind eben keine Word, Excel oder PowerPoint-Dateien, sondern haben die Endungen .html, .css und .js. Solche Dateien kannst du in einem sogenannten Editor erstellen. Ein Editor ist ein Programm, in das du - ähnlich wie in ein Word-Dokument - Text schreiben kannst. Das Tolle an so einem Editor ist, dass er extra dafür entwickelt wurde, Code zu schreiben. Er unterstützt dich beim Programmieren, indem er deinen Code zum Beispiel unterschiedlich einfärbt. Es gibt verschiedene dieser Editoren, wie zum Beispiel Atom oder Visual Studio Code. Wenn du bei einem Entwickler ein schwarzes Fenster mit buntem Text auf dem Laptop siehst, könnte das sein Editor sein.

Du erstellst also eine solche Datei, speicherst sie in deiner Ordnerstruktur und schreibst dann mithilfe deines Editors deinen Code in die Datei. In eine HTML-Datei schreibst du den Inhalt, den deine Seite zeigen soll. Allein der HTML-Code lässt deine Seite aber ziemlich traurig aussehen. Du wirst, wenn du sie im Browser öffnest (das kannst du tatsächlich machen, indem du einfach deine HTML-Datei oben in dein Browserfenster ziehst), einfach nur deinen lieblos heruntergeschriebenen Text, Links und vielleicht ein paar Bilder sehen. Um das Ganze hübsch zu machen, deine Inhalte schön anzuordnen, ihnen Farbe und Ästhetik zu verleihen, schreibst du deinen CSS-Code. Und wenn du nun noch Interaktionen hinzufügen möchtest, dann kommt JavaScript ins Spiel. Mit JavaScript kannst du zum Beispiel erreichen, dass deine Seite von selbst wieder an den Anfang scrollt, wenn du auf ein bestimmtes Symbol klickst.

Alles, was du auf meiner Seite an Inhalt lesen kannst, habe ich in eine HTML-Datei geschrieben. Zum Beispiel diesen Post, aber auch mein Menü, meine Fußleiste usw. Die Position der einzelnen Texte und Bilder, meine Schriftarten und Farben habe ich mit CSS definiert. Und die Slideshow auf meiner Startseite funktioniert durch einen kleinen Code in meiner JavaScript-Datei.

Programmiersprachen werden jeweils mit ihrer ganz eigenen Grammatik und ihrem eigenen Vokabular geschrieben. Nur so versteht der Browser, wenn er eine Webseite lädt, wie sie aussehen soll. Und genau dieses Schreiben des Codes nach all seinen Regeln und das Verknüpfen der Dateien untereinander ist die ganze Zauberei. Auf die einzelnen Sprachen werde ich nochmal genauer in eigenen Blog-Posts eingehen. Schaue also gerne rein, wenn du mehr dazu wissen willst.

Nur weil ich meinen Code geschrieben habe, heißt das aber nicht, dass du meine Webseite direkt besuchen kannst. Denn meine Dateien liegen momentan noch lokal auf meinem Laptop. Das bedeutet, ich kann mir meine Seite zwar in meinem eigenen Browser ansehen, alle anderen Menschen können das aber nicht. Wie man eine Seite ins Internet bekommt, ist nochmal ein anderes Thema. Und dazu kannst du hier mehr lesen.

Ich hoffe, dir hat der Artikel gefallen!
Bis zum nächsten Mal, ich freue mich auf dich!
